01

Overview

Add the ability to call arbitrary OpenRouter-hosted models from inside Claude Code workflows, exposed as a single MCP primitive. Two usage patterns build on it: multi-model council lenses for genuine cross-vendor deliberation, and cheap sub-task routing that reserves Opus for synthesis.

02

Goals & non-goals

Goals

  • One reusable consult primitive — main loop, subagents, and Workflow agents.
  • Lenses reuse existing persona prompts verbatim; diversity comes from the model.
  • OpenRouter is never a hard dependency — any failure falls back to Claude.

Non-goals

  • Pointing the whole harness at OpenRouter via ANTHROPIC_BASE_URL.
  • Multi-turn agentic loops on non-Claude models — consult is single-shot.
  • Tool use / function calling on non-Claude models (future extension).
03

Architecture

A small openrouter MCP server registered in settings.json → mcpServers. Python, scaffolded by the new-mcp-server skill. Reads OPENROUTER_API_KEY from env; never committed. Maps to POST /api/v1/chat/completions, sets attribution headers, and fails soft so callers can detect errors and fall back.

Routing config — skills/council/model-routing.json

{
  "lenses":   { "<lens>": "claude" | "<openrouter-model-id>" },
  "tasks":    { "classification": "<cheap-model>", "scout-research": "…" },
  "defaults": { "lens": "claude", "fallback": "claude" }
}
04

The two patterns

Pattern A

Multi-model council lenses

A non-Claude lens is dispatched as a thin Claude relay agent(): it calls consult(model, system=<persona>, prompt) and returns the model's text verbatim. The Workflow sandbox can't make HTTP calls itself, but a Claude agent it spawns can reach the MCP tool — so deterministic orchestration stays intact and the lens output is genuinely another vendor's reasoning.

Pattern B

Cheap sub-task routing

Skills and subagents that run a routed task class — classification, scout research, scoring — call consult(<cheap-model>, …) directly. No relay needed: main-loop and subagent contexts already have MCP access. On failure, fall back to the normal Claude path for that sub-task.

05

Resilience & testing

Degrade gracefully

Every caller treats a consult error — missing key, 4xx/5xx, timeout — as “use Claude instead.” An OpenRouter outage degrades to today's all-Claude behavior. OPENROUTER_API_KEY lives in the shell env only; the server fails soft and logs once if it's absent.

Verify

  • Unit — mock OpenRouter; assert request shape, parsing, error→fallback.
  • Smoke — one live call to a cheap model behind a flag.
  • Integration — quick-mode council run with one non-Claude lens.

Implementation

Phase 1 · lower risk

MCP server + Pattern B

Build the openrouter MCP server, the routing-config loader, and cheap-routing for one sub-task class. Exercises the primitive end-to-end and is immediately useful.

Add the relay in council-deliberate.js and the lenses routing table; verify diversity in a deliberation run.
