# Oracle Final Position — Claude Config & Model-Routing Optimization

**Revised recommendation:** Keep the ONE unified routing system keyed by `(spawn-site × profile × account-type)` in `skills/council/model-routing.json`, but move the eval gate from "before everything" to **activation boundaries**. The v1 MVP (F1 alias fix, F2 permissions, F4 stale-ID refresh, F5 routing-design doc, plus the split-out F3a fail-soft hook wrapper) changes no live model behavior and ships ungated. A **12-case smoke eval** (F11) becomes the merge gate for the first `routed_consult` caller and any live model flip in `/ship` or council profiles. The **full 38-case golden harness** slides to v1.1 and remains the mandatory entry gate for F6 (OpenRouter lens relay) and F7 (evidence-based skill pruning). On the Max plan, cheap-task routing stays **in-family Haiku 4.5** (free under plan, no key, no cross-vendor egress); OpenRouter spends dollars only on the API account and only on tasks an eval clears.

**Concessions made:**
- **Eval-before-everything → eval-before-activation.** I conceded in the Challenge round that F1/F2/F4/F5 flip zero live model behavior and F4's stale IDs have zero callers, so gating the quick-win PR on a golden harness was misapplied rigor and pure schedule drag. The harness attaches where behavior actually changes, not where config text moves.
- **Smoke-set sizing.** Accepted Strategist's split: a 12-case smoke set (not the full 38) is the proportionate gate for the first live caller, shipped as an acceptance criterion inside F5 (~0.5 day). The 38-case harness is reserved for the higher-stakes boundaries (relay, pruning).
- **F3a coupling.** Accepted Operator/Strategist's F3 split. F3a (fail-soft existence guard + vendored wrapper, removing the private path from public `settings.json`) joins v1; F3b (per-call interpreter perf, batching, alert tuning) defers to v1.1. My own routing metrics and any evidence-based prune depend on this telemetry pipeline surviving a clean clone, so I actively want F3a in the MVP.
- **Registry data is inconclusive, not evidence.** Conceded to Craftsman/Scout that all nonzero registry uses are this session's own uncommitted writes and committed HEAD shows 0/67. I will NOT treat zero-uses as a prune signal. Council-skill pruning waits for a 2-4 week window of durably-incremented data; dormant-suite extraction proceeds on workload judgment, independent of the registry.

**Non-negotiables:**
1. **No live model flip without a passing smoke eval.** The moment `routed_consult` gets its first real caller, or `/ship`/`/looper`/council profiles change which model runs, the 12-case smoke set (composite ≥ 4.0, format compliance 100%) must pass in CI. "The demo looked fine" is not a merge signal.
2. **No Phase 2 lens relay, and no council-skill prune, without the full 38-case harness passing.** These are the two decisions with the largest blast radius and the least reversibility; both gate on the golden set.
3. **Pattern A (OpenRouter lens relay) must prove measurable diversity value or collapse to Pattern B (API-account-only, no relay).** A Claude-tuned persona reused verbatim on a non-Claude model is an unproven transfer. The gated trial MUST run each candidate lens persona through the golden set on both Claude and the target vendor model, log/assert the ACTUAL vendor model returned, and only promote lenses whose non-Claude score is within threshold. If diversity value is not measurable, OpenRouter routing is cost-only and belongs on the API account exclusively. This aligns with Skeptic's send-allowlist + ZDR/no-training + kill-switch conditions.
4. **No pinned `claude-*` model IDs as defaults, and no config self-contradiction.** `claude-fable-5[1m]` must revert to a tier alias, and the `[1m]` suffix reconciled with `CLAUDE_CODE_DISABLE_1M_CONTEXT=1` (drop one or the other; a governance repo cannot ship a default that fights its own env flag and its own alias-only rule).
5. **Structured cheap-task output is validated before trust.** Any classification/scoring task routed to a cheap model must return schema-constrained output and pass a rule-based validator in the caller. Never trust a routed model's structured output unverified.

**Implementation notes:**
- **Golden harness location:** `pipeline/evals/` (outside skill dirs per governance spec). Smoke set = 12 cases (representative classification / scoring / scout / council-round / implement / review). Full set = 38 (≈20 common, ≈10 edge, ≈8 adversarial incl. prompt-injection in retrieved text). Composite gate: `correctness (LLM-judge ≥4.0) × 0.45 + format 100% × 0.25 + persona-fidelity ≥4.0 × 0.20 + cost/latency-within-profile × 0.10`.
- **Max-plan routing:** cheap-task class → `claude` → `claude-haiku-4-5-20251001`, temp 0. Session default → `opus` daily, `fable` for ceiling; drop `[1m]`. Council/audits unchanged (already correct). `/ship` implement → `opus`; PR review → `fable`/`opus`.
- **API-account routing:** session default `sonnet`, escalate manually; `/ship` implement `sonnet`, review `sonnet`→`opus` on repeat failure; OpenRouter cheap tasks → a current-gen cheap model (F4 refresh replaces stale `openai/gpt-4o-mini` / `google/gemini-flash-1.5`; pick concrete IDs at implementation from the claude-api skill / OpenRouter `list_models`), eval-gated at temp 0.
- **Telemetry hardening (feeds prune evidence):** F3a wrapper fail-soft on missing private hook; extend `registry.json` increment to log routed-model + sampled score per spawn so version drift becomes visible instead of the current all-zeros blind spot. Durable increments (untracked sidecar or auto-commit) per Craftsman so the 2-4 week window yields real data.
- **Prompt versioning:** council personas ARE prompts but lack a changelog linked to eval scores; add one before any persona is promoted to a non-Claude model.
- **Sequencing summary:** v1 = F1, F2, F3a, F4, F5 (+12-case smoke acceptance criterion). v1.1 = F3b, F11 full harness, then F6 relay trial and F7 prune, both gated on the 38-case set.
