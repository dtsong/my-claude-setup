Tension ledger

Four pairings went to adversarial challenge on Fable. Every verdict below changed the plan.

VS Eval harness first, or MVP now
Oracle: a 38-case golden harness must exist before any re-routing or pruning; without it, model swaps silently degrade quality.
Verdict

Eval-before-activation. MVP ships ungated; a 12-case smoke set gates the first live model flip; the 38-case harness gates relay and prune.

Strategist: the harness ranks nowhere in the MVP; config hygiene and the routing table are RICE 240-2000 quick wins that should ship now.

Both moved. F1/F2/F4/F5 change no live model dispatch, so gating them was misapplied rigor; the gate belongs where behavior actually changes.

VS Is the usage registry a trustworthy instrument
Craftsman: zero uses is a measurement artifact. The telemetry script is unwired and watches the wrong layer; council skills load by inlined file reads it never sees.
Verdict

Craftsman was right; Scout and Operator retracted. Committed HEAD shows 0/67 uses; every nonzero entry was this session's own uncommitted write.

Scout + Operator: 10 of 67 skills show real uses; tracking works, so instrument for 2-4 weeks and prune on the resulting evidence.

Consequence: no pruning on current data. Make the increment durable and deterministic first; zero-use is inconclusive, any-use is keep.

VS OpenRouter egress: block or trial
Skeptic: consult() forwards system + prompt verbatim to OpenAI and Google with no redaction or allowlist. Lens relay over repo content is uncontrolled egress across the trust boundary: Critical.
Verdict

Gated trial, one lens only: send-allowlist, ZDR + no-training flags, actual-model audit assertion, kill switch. Prove measurable diversity or collapse to Pattern B, API-only.

Scout: keep OpenRouter for cross-vendor lens diversity (Pattern A) and for cheap routing on the API account after refreshing the stale IDs.

The verbatim-forwarding finding held under re-verification, but platform controls address the controllable half; the gated trial doubles as Oracle's persona-transfer experiment.

VS Telemetry hook fix: v1 blocker or v1.1 fill-in
Operator: five lifecycle events bare-call a private-repo script on every tool use. It breaks every fresh clone and the API work account, and leaks a private path through public config: ship-blocker.
Verdict

Split. F3a (fail-soft dispatcher + path removal, XS effort) joins v1 and goes first in the ship order. F3b hardening defers to v1.1.

Strategist: demote to v1.1 fast-follow; the MVP should stay the four config-hygiene and routing wins.

The ship-blocking piece is five string edits plus a five-line script; Strategist's own RICE table had F3 above F5, so the original cut-line was inconsistent.

Risk register

Rollback posture: git checkout settings.json takes effect immediately through the symlink; OpenRouter dies via an empty tasks map; extraction reverts as one commit.

Critical if unconstrained

OpenRouter egress

Verbatim system + prompt forwarding, zero redaction. No routed egress without send-allowlist, ZDR + no-training flags, actual-model assertion, and a kill switch.

High

Telemetry blast radius

Every tool call cold-starts python3 from a private repo path. Any clone without it errors on every call. F3a is the v1 ship-blocker.

High

Live symlinked config

settings.json is production config with no staging. A malformed edit breaks the running harness. Pre-commit JSON-schema validation is an acceptance criterion.

High

1M-context contradiction

The [1m] suffix requests a variant that DISABLE_1M_CONTEXT=1 removes. Resolve to one deliberate state before F1 lands.

High

API runaway spend

A routing typo on the dollar-billed account has no cap. Per-model max_tokens clamp, frontier-ID denylist, and volume alerts.

Medium

Extraction dangling refs

Non-atomic ECE moves break /ece, registry entries, and cross-links. Grep all refs, single commit, manifest, reference-integrity hook.
